1. О параметрах в файлах свойств

Для уменьшения дублированности кода:

Из файлов resources\db\**\*.properties можно было-бы выделить одинаковые свойства и сделать файл common.properties,
а то, что кастомизируется для конкретной БД - оставить в кастомных файлах по типу СУБД.
(Сделано 07.05.2025).

2. О SQL

Для уменьшения дублированности кода:

Ну и в идеале из двух resources\db\**\initDB*.sql сделать на каком-то универсальном диалекте SQL.
И если необходимо, применить какой-то транслятор для перевода sql в диалект конкретной СУБД.
